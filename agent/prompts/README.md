# Guardian Prompts Directory

This directory contains Copilot-ready prompts generated by the Project Guardian agent.

## Directory Structure

```
prompts/
├── chat/       # Prompts optimized for VS Code Copilot Chat
├── cli/        # Prompts optimized for GitHub Copilot CLI  
├── batch/      # Combined prompts for batch processing
└── INDEX.md    # Auto-generated index of all prompts
```

## Usage

### VS Code Copilot Chat

1. Open a prompt file from `chat/` directory
2. Copy the entire content
3. Paste into VS Code Copilot Chat
4. Review and apply the suggested fix

**PowerShell shortcut:**
```powershell
Get-Content "agent/prompts/chat/<filename>.md" | Set-Clipboard
```

### GitHub Copilot CLI

```powershell
# Using gh copilot suggest
Get-Content "agent/prompts/cli/<filename>.md" | gh copilot suggest

# Or copy to clipboard
Get-Content "agent/prompts/cli/<filename>.md" | Set-Clipboard
```

### Batch Processing

Batch prompts combine multiple issues of the same category for efficiency.
Use these when fixing multiple related issues at once.

## Generating Prompts

```powershell
# Generate prompts for all issues
npm run guardian -- --generate-prompts

# Generate prompts for specific task
npm run guardian -- --task "fix typescript errors"

# Interactive mode
npm run guardian -- --interactive
```

## Prompt Format

Each prompt includes:
- **Problem**: Clear description of the issue
- **Location**: File path and line number
- **Source Context**: Relevant code with highlighting
- **Project Conventions**: Guidelines to follow
- **Expected Outcome**: Success criteria

## Managing Prompts

```powershell
# List all prompts
npm run guardian -- --list-prompts

# View a specific prompt
npm run guardian -- --show-prompt <filename>

# Clear all prompts
npm run guardian -- --clear-prompts
```

## Tips

1. **Chat prompts** are more detailed - use for complex issues
2. **CLI prompts** are concise - use for quick fixes
3. **Batch prompts** save time when fixing multiple similar issues
4. Always review AI suggestions before applying
5. Run tests after applying fixes: `npm test`
